# Бриф: Топ-3 из экспертного анализа

## Что делаем и зачем

По результатам экспертного анализа (analysis-results.md) реализуем 3 фичи с максимальным эффектом на обучение:

1. **Режим «Микс дня»** — interleaving (смешанная практика) из разных модулей
2. **Bias к пограничным ситуациям** — больше трудных, обучающих рук
3. **SM-2 интервальное повторение** — ошибки возвращаются через 1/3/7/14 дней

---

## Таск 1: «Микс дня» — новый тренажёр

### Что
Новый модуль `js/trainers/daily-mix.js` — кнопка на welcome screen + вкладка в tab bar. Генерирует сессию из 10 задач, чередуя модули (interleaving).

### Логика выбора задач
1. Получить getAllStats() из ProgressTracker
2. Выбрать 10 задач из разных модулей, приоритизируя слабые (accuracy < 80%)
3. Микс должен содержать задачи из минимум 3 разных модулей
4. Порядок — чередование (не 5 opening подряд, потом 5 response)

### Типы задач в миксе
Для каждого модуля генерировать задачу как в соответствующем тренажёре:
- **hcp**: генерировать руку, спросить HCP
- **opening**: генерировать руку, спросить открытие
- **response**: генерировать ситуацию, спросить ответ
- **conventions**: Стейман/Блэквуд/контра — случайный
- **lead**: контракт + рука, спросить карту
- **quiz**: случайный квиз из quizzes.js

### UI
- На welcome screen: большая кнопка «Микс дня» сверху (заметная, accent цвет)
- В tab bar: новая вкладка (первая после welcome)
- Экран: stats bar + задача + кнопки ответов + feedback + «Следующая»
- После 10 задач: итоговый экран с результатами сессии
- Паттерн UI — как в существующих тренажёрах (renderStats, renderHand, feedback-area)

- Изменить: `js/app.js` — добавить в MODULE_LOADERS и MODULE_TITLES
- Изменить: `index.html` — добавить tab-item и кнопку на welcome

## Таск 2: Bias к пограничным ситуациям

### Что
Изменить `js/core/dealer.js` — функции `dealForOpening()` и `dealForResponse()` должны чаще генерировать пограничные руки.

### dealForOpening() — пограничные случаи (добавить 30% вероятности)
- **11-12 HCP** — граница открытия (открывать или пас?)
- **15 HCP с 5-карточным мажором** — 1БК или 1♠/1♥?
- **18-19 HCP равномер** — 1БК (18) или 2БК (19)?
- **21-22 HCP** — обычное открытие или 2♣ ФГ?

### dealForResponse() — пограничные случаи
При генерации руки South (отвечающего):
- **5-6 HCP** — пас или простой подъём?
- **9-10 HCP с фитом** — подъём 2M или инвит 3M?
- **12-13 HCP** — инвит или гейм?
- **7-8 HCP после 1БК** — пас или Стейман?

### Реализация
Перед текущей логикой добавить: `if (Math.random() < 0.3) { /* пограничный случай */ }`. Не ломать существующую логику — пограничные случаи добавляются, а не заменяют.

## Таск 3: SM-2 интервальное повторение

### Что
Расширить `js/progress/tracker.js` — добавить систему интервального повторения ошибок.

### Модель данных SM-2
В localStorage добавить отдельный ключ `bridge-trainer-sm2`:
```javascript
{
  items: [
    {
      id: "opening:12hcp-5spades-balanced",  // уникальный тип ситуации
      module: "opening",
      description: "12 HCP, 5♠, равномер — открытие?",
      lastReview: 1711612800000,  // timestamp
      interval: 3,  // дней до следующего повторения
      easeFactor: 2.5,  // множитель SM-2
      repetitions: 2,
      nextReview: 1711872000000  // когда показать снова
    }
  ]
}
```

### API (добавить в ProgressTracker)

```javascript
// Записать ошибку для SM-2
ProgressTracker.recordError(module, situationId, description)

// Записать правильный ответ для SM-2 (увеличить интервал)
ProgressTracker.recordSuccess(situationId)

// Получить задачи, которые пора повторить (nextReview <= now)
ProgressTracker.getDueItems() → [{id, module, description, ...}]

// Получить количество задач на повторение
ProgressTracker.getDueCount() → number
```

### Алгоритм SM-2 (упрощённый)
```
При ошибке (quality < 3):
  interval = 1 день
  repetitions = 0

При правильном ответе (quality >= 3):
  if repetitions == 0: interval = 1
  if repetitions == 1: interval = 3
  if repetitions == 2: interval = 7
  else: interval = round(interval * easeFactor)

  easeFactor = max(1.3, easeFactor + 0.1 - (5-quality) * (0.08 + (5-quality) * 0.02))
  repetitions++

nextReview = now + interval * 86400000
```

### Интеграция в «Микс дня»
В daily-mix.js: перед генерацией новых задач проверить `getDueItems()`. Если есть задачи на повторение — включить их в микс ПЕРВЫМИ (до 5 из 10).

### Интеграция в welcome screen
Показать бейдж: «3 задачи на повторение» рядом с кнопкой «Микс дня».
