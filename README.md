# Тренажёр спортивного бриджа

Интерактивное веб-приложение для отработки навыков спортивного бриджа. Покрывает все темы из 10 занятий Федерации спортивного бриджа России.

**Live:** [alexagula.github.io/bridge-trainer](https://alexagula.github.io/bridge-trainer/)

## Что умеет

| # | Модуль | Описание |
|---|--------|----------|
| 1 | Подсчёт HCP | Оценка силы руки на время (5/10/15/30 сек), очки за расклад |
| 2 | Открытие | Выбор открытия по 7-шаговому алгоритму (пас, 1М, 1БК, 1м, 2♣, 2БК, блоки) |
| 3 | Ответ | Ответ на открытие партнёра (1М, 1БК, 1м, 2♣, 2БК, слабые 2) |
| 4 | Торговля | Полная симуляция торговли за 4-х игроков — вы играете за Юг |
| 5 | Конвенции | Стейман, Блэквуд, вызывная контра — отдельные упражнения |
| 6 | Розыгрыш | Готовые сценарии: импас, экспас, убитка, развитие масти |
| 7 | Первый ход | Выбор карты для атаки по правилам виста |
| 8 | Тесты | Правда/Ложь по материалам всех 10 занятий |
| 9 | Справочник | Поиск по терминам, фильтр по занятиям |
| 10 | Прогресс | Статистика по каждому модулю, серии, точность |

## Технологии

- **Чистый HTML + CSS + JS** (ES modules) — без фреймворков и зависимостей
- **PWA** — устанавливается на телефон, работает офлайн
- **Mobile-first** — адаптивный интерфейс, тёмная тема
- **localStorage** — прогресс сохраняется на устройстве
- Карты рисуются через CSS (без картинок)
- Правила бриджа закодированы как JS decision trees

## Запуск локально

Любой HTTP-сервер из корня проекта:

```bash
# Python
python3 -m http.server 8000

# Node.js
npx serve .

# VS Code
# Установить расширение "Live Server" → правый клик на index.html → Open with Live Server
```

Затем открыть `http://localhost:8000` в браузере.

> Просто открыть `index.html` как файл не получится — ES modules требуют HTTP-сервера.

## Принципы

- **Русская терминология** — Т/К/Д/В (не A/K/Q/J), масти ♠♥♦♣
- **Правила из занятий** — источник истины: PDF-файлы Федерации
- **Генерация рук** — rejection sampling с ограничениями (HCP, расклад, масти)
- **Торговля как код** — decision trees с пошаговым объяснением каждого решения
- **Сценарии розыгрыша** — предзаготовленные ситуации вместо полного игрового движка
