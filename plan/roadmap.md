## Что уже сделано (v1.0)

Приложение полностью собрано и функционально. Всё ниже — работающий код.

### Тренажёры (10 модулей)

| # | Модуль | Файл | Что делает |
|---|--------|------|------------|
| 1 | HCP Trainer | hcp-trainer.js | 5 типов вопросов (HCP, дистрибуция, баланс, форма), таймер, режимы базовый/расширенный |
| 2 | Opening Trainer | opening-trainer.js | Генерация рук → выбор открытия, дерево решений из 7 шагов с объяснениями |
| 3 | Response Trainer | response-trainer.js | 9 типов открытий (1♥, 1♠, 1БК, 1♣, 1♦, 2♣, 2БК, 2♥, 2♠), фильтры |

| 5 | Convention Drill | convention-drill.js | Стейман (ответ на 2♣), Блэквуд (ключевые карты), вызывная контра |
| 6 | Play Trainer | play-trainer.js | Сценарии: импас, экспас, убитка. Фильтр по категориям |
| 7 | Lead Trainer | lead-trainer.js | Выбор карты первого хода, clickable карты, секвенции, 4-я сверху |
| 8 | Quiz Trainer | quiz-trainer.js | 57 true/false квизов по всем 10 занятиям, фильтр по урокам |
| 9 | Theory Reference | theory.js | Справочник терминов и правил с поиском, фильтр по занятиям |
| 10 | Progress | tracker.js | Статистика по всем модулям, точность, серии, среднее время |

### Инфраструктура

- PWA: manifest.json + service-worker.js (v4, cache-first, офлайн)
- Mobile-first тёмная тема, tab-навигация
- SPA с lazy-loading модулей
- Прогресс в localStorage (до 500 результатов на модуль)

## Что можно делать дальше

Приложение работает. Дальнейшее развитие зависит от реального использования. Ниже — направления, отсортированные по ценности. **Ни одно не обязательно — всё уже работает.**

### Направление 1: Тестирование и деплой

Приложение ни разу не деплоилось и не тестировалось на реальном устройстве.

- [ ] Задеплоить на GitHub Pages
- [ ] Протестировать на телефоне (iOS Safari, Android Chrome)
- [ ] Проверить офлайн-режим (PWA)
- [ ] Проверить все 10 модулей — найти баги
- [ ] Исправить найденные баги

**Когда:** сейчас. Это первый шаг перед любым развитием.

---

### Направление 2: Больше контента

Движок работает, но контента может не хватать для длительного использования.

- [ ] Больше сценариев розыгрыша (сейчас ~10 в techniques.js — импас, экспас, убитка)
- [ ] Больше квизов (сейчас 57 — можно расширить до 100+ из материалов Quizlet mega pack)
- [ ] Больше записей в справочнике теории (theory-entries.js)
- [ ] Сценарии на защиту (сигнализация, подкладки)

**Когда:** после того как сам прорешал все существующие задачи и чувствуешь нехватку.

---

### Направление 3: Качество тренировки

Улучшить то, как приложение обучает.

- [ ] SM-2 интервальное повторение — ошибочные задачи появляются чаще
- [ ] Режим «слабые места» — тренировка только по модулям с точностью < 80%
- [ ] Адаптивная сложность — если точность > 90%, давать пограничные руки
- [ ] Ежедневная 10-минутная сессия — приложение само выбирает микс тем
- [ ] Статистика по дням — график тренировок, тренд

**Когда:** после 100+ решённых задач, когда понятно, что именно проседает.

---

### Направление 4: Полировка UX

- [ ] Тактильная обратная связь (вибрация на правильный/неправильный)
- [ ] Анимации переходов между задачами
- [ ] Звуковые эффекты (опционально)
- [ ] Онбординг для нового пользователя
- [ ] Экспорт прогресса (JSON/CSV)

**Когда:** когда приложение используется регулярно и хочется комфорта.

---

### Направление 5: AI-тренер (Claude)

- [ ] Proxy для API-ключа (Cloudflare Worker — ключ не на клиенте!)
- [ ] Кнопка «Объясни подробнее» — Claude разворачивает объяснение
- [ ] Анализ паттернов ошибок каждые 20 задач
- [ ] Персональные рекомендации по тренировке

**Когда:** только если объяснения из дерева решений перестают хватать. API-ключ — через proxy, никогда в клиентском JS.

## Что НЕ стоит делать

| Идея | Почему нет |
|------|-----------|
| Live Game с AI-оппонентами 3 уровней | Огромный scope. Bidding Simulator уже даёт полный цикл торговли. Розыгрыш ходом за ходом — отдельный проект |
| Фиксированный банк 806 задач из tasks_v3.ndjson | Движок генерирует бесконечные задачи. Фиксированный банк — регресс |
| rules.ndjson как источник истины | opening.js и response.js уже содержат те же правила в коде с объяснениями |
| Мультиплеер | Преждевременно. Нет подтверждённого спроса |

## Принципы

1. **Сначала используй, потом улучшай.** Задеплой → потренируйся неделю → запиши что бесит → чини это.
2. **Код = правила.** opening.js и response.js — источник истины. Не дублировать в JSON.
3. **Генерация > фиксированный банк.** Движок создаёт бесконечные задачи.
4. **Безопасность.** API-ключ никогда в клиентском JS.
5. **Не добавляй фичи, пока не кончился контент.** 10 тренажёров, 57 квизов, справочник — это много. Сначала прорешай.
