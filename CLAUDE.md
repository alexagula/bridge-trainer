# Проект: Тренажёр спортивного бриджа

## Статус: задеплоен и работает

**Live:** https://alexagula.github.io/bridge-trainer/
**Репо:** https://github.com/alexagula/bridge-trainer
**Стек:** чистый HTML + CSS + Vanilla JS (ES modules), без фреймворков. PWA.

---

## Контекст

Автор — предприниматель, учится играть в спортивный бридж. Прошёл 10 занятий в Федерации спортивного бриджа России (PDF-файлы в папке Lessons/).

Приложение помогает:
1. Запомнить правила и термины из всех 10 занятий
2. Автоматизировать принятие решений (оценка руки, открытие, ответ, вход)
3. Практиковаться в реальных ситуациях

## Архитектура тренажёров

Все тренажёры следуют одному паттерну:

```
class XxxTrainer {
  constructor(containerId) — привязка к DOM
  init()                   — render() + первая задача
  destroy()                — очистка (таймеры и т.д.)
  render()                 — HTML + обработчики
  newProblem()             — генерация задачи
  checkAnswer()            — проверка + feedback + запись в ProgressTracker
}
```

Новый тренажёр добавляется за 3 шага:
1. Создать файл в js/trainers/
2. Зарегистрировать в MODULE_LOADERS в app.js
3. Добавить tab в index.html

## Правила для Claude

1. **Источник истины для правил бриджа** — PDF-файлы в Lessons/. Если правило есть в PDF — следуй ему.
2. **Источник истины для кода** — JS-файлы в js/. Не дублировать правила в JSON-файлах.
3. **Терминология** — русские обозначения: Т/К/Д/В (не A/K/Q/J), масти ♠♥♦♣.
4. **Уровень** — автор новичок после 10 занятий. Не перегружать теорией.
5. **Стиль** — коротко, по делу. Код > теория.
6. **Безопасность** — API-ключи только через proxy/env, никогда в клиентском JS.
7. **Паттерн** — новые тренажёры добавлять по существующему паттерну (см. hcp-trainer.js как образец).
